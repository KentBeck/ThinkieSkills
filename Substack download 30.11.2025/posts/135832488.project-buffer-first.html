Pattern: You’re starting to plan a project with a fixed delivery date

Transformation: Start by saying, “How certain do we need to be about the scope & date? Okay, we’ll start with 15% (or 10% or 20%) project buffer.”

This Thinkie seems to be terrifying to folks. The planning process seems like it’s supposed to be based on over-promise/under-deliver. I hate working under a cloud of failure, though. Starting with a project buffer lets the team work with the assumption of success.

Tetris projects are where you take all the available time & fill it with vaguely defined & uncertainly estimated features. Think about the last row of features you add. How certain are you of those decisions—the scope, cost, & priority of the features?

You don’t lose any value not making those decisions today. Instead the last row is a reassuring block that says “Buffer”.
