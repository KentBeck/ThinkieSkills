Pattern: You are stuck & overwhelmed trying to solve a big problem.

Transformation: What thing would, if you had it, make the big problem more tractable? Make that.

In a way Scale Down seems like it’s too simple to need naming & explaining. However, when I’m in that moment of overwhelm, that’s exactly when I need the reminder that it’s okay to solve a part of the problem.

And it’s not just any old part of the problem that called forth by Scale Down, it’s specifically a part of the problem that will make the overall problem easier to solve. “Well, I’ve written one line of code so that’s one less line of code I’ll need to write.” Nope. There aren’t 1000 lines of code to be written. There are <you don’t know how many> lines of code to be written. And you’re feeling overwhelmed.

“I need to format this data as JSON & send it to the server.” Okay, get “format this data as JSON” working & then the remaining slice of cake will be smaller.

Here are some criteria you could use to select the Scale Down:

  • Something you understand well already so you can get some momentum.

  • Something that you need to interact with another team to accomplish. It’s good to do enough work together that you can then proceed independently.

  • Something you don’t understand at all. Knocking this off will buy you big confidence.

  • Something that can already provide customer value. Perhaps you’re working on a calculation with a fancy UI. Implement the calculation & surface it as a spreadsheet macro first.

  • Something that’s kind of a cruddy job.

I’m not sure why I resist Scale Down, but I sure do. I think I retain some amount of shame that I can’t just solve the whole thing all at once. If I was a Real Programmer™️, I could sit down, type really really fast for an uninterrupted hour, & be done. To the amazement of colleagues & onlookers.

The first step of Scale Down for me is letting go of that shame. Then I can ask, “What is a thing (object, function, service, whatever) which, if I had it, would bring the overall solution closer?”
