This one wins the prize for simplest tidying. You're reading a big chunk of code & you realize, "Oh, this part does this and then that part does that." Put a blank line between the parts.

I like that this tidying is so so simple. That's part of the philosophy of Tidy First?--don't make software design such a big deal that you are in danger of not doing it.

Software design enables change. A little software design can make change a little easier.

Here's the cool thing--compound interest. Software design also makes more software design easier. This is both a blessing and a curse. You can get caught in the vortex of design & forget to make the change. So don't do that (we'll talk about the Design Vortex in the next section). Done well, software design enables software design that enables change.
