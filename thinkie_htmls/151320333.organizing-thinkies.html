1. Learning Path (Pedagogical Organization)

Foundation Level: Core Concepts and Simple Applications

  1. Basic Pattern Recognition

    • Can't/Because (converting blockers to conditions)

    • "Obviously..." (challenging assumptions)

    • Laughter (noticing suppressed insights)

    • Reverse Phrase (language pattern transformation)

  2. Simple Problem Reframing

    • Easy Bit (smallest helpful action)

    • Fun Bit (enjoyable starting point)

    • Naive Solution (questioning complexity)

    • Scale Down (reducing scope strategically)

  3. Basic Decision Making

    • Force Decision (breaking paralysis)

    • Defer Decision (waiting purposefully)

    • McDonald's (catalyzing decisions)

    • Upside/Downside (quick consequence analysis)

Intermediate Level: System Understanding

  1. Process Analysis

    • Work/Wait Ratio (measuring process efficiency)

    • Match the Pipes (balancing capacity)

    • Latency, Throughput, & Variance (understanding performance dimensions)

    • Special-Cause Versus Common-Cause Variation (classifying problems)

  2. Project Management

    • Half in Half (scope reduction)

    • Project Buffer First (planning with safety)

    • One Pass Quickly, One Pass Good (staged improvement)

    • End To End (early integration)

  3. Feedback Systems

    • Direct Feedback (removing layers)

    • Personal Feedback (self-usage)

    • Customer's Voice (unified input)

    • Put Skin In The Game (aligning incentives)

Advanced Level: Complex Systems

  1. Strategic Thinking

    • Growth Rate, Not Absolute Number (trend analysis)

    • Rate-Limiting Resource (constraint management)

    • Extrapolate Further (trend projection)

    • Trust the Market (understanding complex systems)

  2. Organizational Design

    • Dependency Merging (structural solutions)

    • Break Dependency (independence creation)

    • Serialize (sequential optimization)

    • Abundance (resource perspective)

  3. Advanced Problem Solving

    • More General Solution (expanding scope)

    • Parallel Decision (multiple paths)

    • Composed Solution (sequential transformation)

    • Hard Problem (comprehensive analysis)

2. Retrieval Framework (Problem-Based Organization)

When You're Stuck

  • Easy Bit (smallest action)

  • Fun Bit (enjoyable piece)

  • Scale Down (reduce scope)

  • Hard Problem (list everything)

  • Naive Solution (question complexity)

When Decisions Are Needed

  • Force Decision (immediate choice)

  • Defer Decision (purposeful waiting)

  • Parallel Decision (multiple paths)

  • McDonald's (catalyze decision)

  • Upside/Downside (quick analysis)

When Systems Are Slow

  • Work/Wait Ratio (process efficiency)

  • Match the Pipes (balance capacity)

  • Reduce Utilization (manage resources)

  • Reduce Variance (stabilize system)

  • Rate-Limiting Resource (manage constraints)

When Dependencies Block Progress

  • Break Dependency (create independence)

  • Dependency Merging (structural solution)

  • Direct Feedback (remove layers)

  • End To End (early integration)

  • Half in Half (scope reduction)

When Change Is Needed

  • "Obviously..." (challenge assumptions)

  • Laughter (notice suppressed insights)

  • Can't/Because (convert blockers)

  • But If You Could... (remove constraints)

  • Perfect Setup (ideal conditions)

When Planning Projects

  • Project Buffer First (start with safety)

  • One Pass Quickly, One Pass Good (staged improvement)

  • Serialize (sequential approach)

  • Abundance (resource perspective)

  • Half in Half (scope management)

Usage Notes

  1. For Learning:

    • Start with Foundation Level

    • Practice each Thinkie before moving up

    • Focus on pattern recognition first

    • Build up to complex applications

  2. For Problem Solving:

    • Use Retrieval Framework

    • Match current situation to category

    • Try multiple Thinkies from relevant category

    • Evaluate results after applying

  3. For Understanding:

    • Study Theoretical Framework

    • Understand mechanisms

    • Look for common patterns

    • Connect to similar techniques

Remember: Thinkies generate ideas, not necessarily good ideas. Always evaluate after generating.
